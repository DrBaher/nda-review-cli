# Changelog

## [Unreleased]

- Added **`negotiate analyze --out-md`** for human-readable markdown post-mortems alongside the JSON dashboard. Includes outcome label + notes, per-round trajectory table, wins-by-party, winner-per-clause, source breakdown, fatigue concessions (flagged for human review), and block diagnosis.
- Added top-level **`Makefile`** with common dev targets: `make test / smoke / build / install-dev / tutorial / clean`. Reduces newcomer cognitive load and gives a single canonical entry point for the local dev loop.
- Added **`.github/dependabot.yml`** for monthly GitHub Actions version bumps (auto-PRs labelled `dependencies` + `github-actions`). Set-and-forget; keeps CI workflow steps from going stale.
- Added **`negotiate validate`** — standalone integrity check (schema version + hash-chain verification + per-round structural shape: round numbering, proposer alternation, signer-matches-proposer, text/text_hash present). Exits 0 on success, 2 on any failure with the specific issue listed. Useful when receiving a state file from another party or after hand-editing.
- CI: added a **`negotiate-smoke`** job that runs the full negotiate flow end-to-end (init → counter → accept → sign-off → finalize → analyze → validate → simulate) on every push/PR. Catches packaging regressions in the negotiate code path.

## [0.5.0] - 2026-05-09

- Added **`non_negotiable_clauses`** in policy: hard floor — clauses in this list are NEVER fatigue-conceded by your agent and are always force-countered when text differs from your preferred, regardless of stance/priority/concession-zone. Provides absolute-redline semantics for clauses you're unwilling to compromise on. Surfaces in the LLM agent's system prompt and the `block_diagnosis` text when the negotiation deadlocks on a non-negotiable conflict.
- Added **`negotiate counter --dry-run`** for previewing the proposal (manual / `--auto` / `--agent`) without writing to the state file. Critical safety for `--agent` mode where you want to see what the LLM proposed before committing.
- Added **`negotiate diff [--from-round N] [--to-round M]`** for clause-by-clause redline view of what changed between two rounds. Optional `--out-md` produces a human-friendly markdown file with code-fenced diff blocks, accepted-clauses list, and fatigue concessions.
- Added **`negotiate analyze`** — read-only post-hoc dashboard for any state file. Reports outcome status, per-round trajectory (agreed/disputed/proposed counts), source breakdown (manual / auto / agent / fatigue), winner-per-clause, fatigue summary, concession trajectory, and a lightweight game-theoretic interpretation (`converged organically` / `converged via fatigue` / `blocked (deadlock detected)` / `withdrawn` / etc.).
- Added **`negotiate withdraw --reason ...`** for graceful exit. Flips status to `withdrawn` (terminal) and blocks further `counter` / `sign-off` / `finalize` commands. Adds `withdrawal: {withdrawn_by, withdrawn_at, reason}` to the state file for audit.
- **LLM agent prompt parity**: the `--agent` mode now receives the same context the deterministic `--auto` mode uses — clause priority ranks, non-negotiable markers (`NON_NEGOTIABLE` / `bounce_count=N`), and behavioural rules in priority order (non-negotiable first, then stance, then priority, then bounce). Brings LLM-mode behavior into line with the deterministic mode, avoiding inconsistencies between the two paths.
- **Tutorial step**: the `tutorial` subcommand now includes a "Concept 4 — Two-party negotiation" step covering stance + priorities + non-negotiable + fatigue + sign-off, plus a "What's next" step with the full negotiate command list.
- Added [`examples/negotiate-cheatsheet.md`](examples/negotiate-cheatsheet.md) — one-page reference covering all 11 negotiate subcommands, the three counter modes, the stance × stance outcome table, terminal statuses, tuning knobs, and common usage patterns.

- Added **fatigue concession** — the deterministic deadlock-breaker for the conservative × conservative stalemate. Each clause has a "bounce count" (consecutive most-recent rounds in which the clause was amended by alternating proposers). Once a clause's bounce count reaches `max_clause_bounces` (default 4 in `config/org-policy.json` `defaults`), the next proposer is forced to concede it regardless of stance, priority, or red flags. The amendment source is tagged `auto:<stance>+fatigue` and surfaced in the sign-off "Fatigue concessions (review carefully)" block. Empirically: cons × cons (which previously blocked) now converges in ~6 rounds with 1 fatigue round; all other stance pairings converge with zero fatigue triggered (they don't need it). `max_clause_bounces: 0` disables fatigue entirely and restores the original blocked-state behavior. Applies uniformly across manual / `--auto` / `--agent` modes — even LLM negotiations converge within bounded rounds.
- Added **clause priorities + logrolling** as the structural fix for the conservative × conservative stalemate. Each party ranks clauses 1..N during `quickstart` (16th question, default = policy clause order); stored as `clause_priorities` in `config/org-policy.json` (always private, never leaves the workspace). The `--auto` agent now combines stance + priorities into a concession zone: conservative concedes its bottom 30% by priority, middleground 60%, compromising 85%. Inside the concession zone, the agent accepts current text regardless of stance; outside, stance's red-flag/diff logic applies. Logrolling resolves clauses where parties' priorities don't overlap — empirically reduces disputes from 3 → 1 in our test fixtures even with conservative × conservative. New `tests/test_negotiate_simulate.py::PriorityLogrollingTests` locks in the behavior.
- Added **`negotiate simulate`** for game-theoretic validation: runs both sides on one machine with configurable stances, max-rounds, and counter mode (deterministic `auto` or LLM `agent`). Emits a structured report with outcome (`converged` / `blocked` / `max_rounds_exceeded`), rounds used, per-round agreed/disputed/proposed trajectory, winner-per-clause (which party's preferred text won), and a `block_diagnosis` listing stuck clauses on stalemate. Confirms the predicted bargaining-theory matrix: `conservative × conservative` blocks (no Nash equilibrium with mutual acceptance under symmetric strict preferences), all other pairings converge in 2–3 rounds.
- Added **stalemate detector** for negotiations: tracks `rounds_without_progress` (consecutive most-recent rounds during which the count of `agreed` clauses did not increase) and flips status to `blocked` once it crosses a threshold (default 4). Subsequent `negotiate counter` calls refuse to advance unless `--force-unblock` is passed. New tests lock in the matrix as a regression.
- Fixed `--auto` acceptance logic: previously only considered the other party's amendments, so an entirely-acceptable initial draft would never be accepted (the clauses had never been "amended"). Now per-clause: counter when stance demands, accept the current text otherwise. This is what makes compromising × compromising converge in 2 rounds instead of looping forever.
- Added **negotiation stance** (`conservative` / `middleground` / `compromising`) — set during `quickstart` (15th question), stored in `config/org-policy.json` `defaults.negotiation_stance`, overrideable per-round with `--stance`. Stance shapes both the LLM agent's system prompt and the new `negotiate counter --auto` deterministic mode that generates amendments from your policy + red-flag detection without an LLM.
- Added **`negotiate sign-off`** as a required human gate before `finalize`. Produces a focused "key points" summary (clauses changed from initial draft, amendments applied with their source — manual / `auto:<stance>` / `agent:<stance>` — and any red-flag patterns still active in the final text), then a single batch Y/N confirmation. `finalize` is now blocked until both parties have signed off; `--skip-signoff` is available for testing/non-binding finalizations.
- Added `negotiate` subcommand suite for two-party turn-taking NDA negotiation. File-based protocol — no networking, state lives in a single JSON document exchanged via any channel (email/Drive/Git). `negotiate init / review / counter / accept / status / sign-off / finalize` cover the full flow. The `--agent --llm` option on `counter` repurposes the LLM-augmented review as a negotiation agent that drafts amendments aligned with each party's policy; humans always sign each round. Per-round SHA-256 hash chain detects tampering on load. `negotiate finalize --to-pdf --sign` invokes user-configured external commands (`docx2pdf_cmd`, `sign_cli_cmd` in `config/integrations.json`) to hand off to your own PDF and signing tooling. New `config/integrations.json.example` ships with the schema and worked examples (incl. LibreOffice and Pandoc fallbacks).
- Added `pyproject.toml` and a `nda-review-cli` console-script entry point — `pipx install .` (or `pipx install git+https://...`) gives you `nda-review-cli` on `$PATH`. Bundled assets (config seeds, templates, fixture NDA) ship inside the wheel.
- Added `__version__` and a `--version` flag on the root parser.
- Bare invocation (`./nda_review_cli.py` with no args) now prints a friendly first-run hint pointing at `tutorial`, `quickstart`, and `setup --quick --yes`.
- Smarter `doctor`: gmail/drive build-playbook paths are no longer hard failures when running in corpus-free mode (no `data/raw_strict` directory). Status downgrades to `skip` with an explanatory note.
- `run_all.sh` and `review_nda.sh` now auto-forward `--why`; they accept optional `LLM=<provider>` and `COUNTERPARTY=<name>` env vars for second-pass LLM and profile-aware review.
- Added [`examples/demo.md`](examples/demo.md) (annotated 60-second walkthrough) and [`scripts/record-demo.sh`](scripts/record-demo.sh) (asciinema recording driver).
- CI: added a `build-and-install-smoke` job that builds the wheel, installs it into a fresh venv, and exercises `nda-review-cli quickstart` + `draft` end-to-end (catches packaging regressions where assets aren't bundled).
- Added opt-in **LLM-augmented review** via `review --llm`. Supports Anthropic, OpenAI, Ollama (local), and any OpenAI-compatible endpoint (Qwen, Together, Groq, vLLM, LM Studio). LLM does three things: votes on each rule-engine finding, adds findings the rules missed, and suggests replacement clause language for high-severity items. Output lands under `llm_annotations` in the review JSON; deterministic findings are never modified. HTTP via stdlib `urllib` — no `anthropic`/`openai` SDK dependency. Provider/model/key configurable via `config/llm.json` (gitignored), env vars (`NDA_LLM_*`), or CLI flags. Per-call confirmation prompt prevents accidental sends; refuses non-interactive sends without `--yes-llm-send` or `NDA_LLM_NO_CONFIRM=1`. SECURITY.md updated with explicit data-flow disclosure.
- Added `draft` subcommand: generate outgoing NDAs (mutual / one-way disclosing) using your house policy clause text. Outputs canonical `.md` plus optional `.docx` (stdlib-only zip-based writer; no `python-docx` dependency). Supports `--template-file` for custom templates with `{{placeholders}}`, `--review-after` to round-trip the draft through `review --why`, and `--no-disclaimer` to suppress the starting-point header. Two bundled templates under `templates/`.
- Added `quickstart` subcommand: 14-question guided setup that wires answers (NDA term length, return-vs-destroy preference, residual-knowledge stance, trade-secret indefinite carve-out, affiliate-disclosure scope) into clause-rule `preferred` text and `red_flags` lists. Writes a replayable `config/quickstart-answers.json`.
- Added `tutorial` subcommand: interactive primer that explains policy/profile/playbook and runs a sandboxed sample review.
- Added human-readable summaries to stderr for `init`, `setup`, `ingest`, and `doctor` (stdout JSON unchanged). Suppressible via `NDA_CLI_QUIET=1`.
- Added top-level `LICENSE` (MIT), `CONTRIBUTING.md`, `SECURITY.md`, `ARCHITECTURE.md`, and a tiered onboarding doc (`GETTING_STARTED.md`).
- Expanded CI matrix to Python 3.9–3.12 with pip caching, added a tutorial smoke job, and added a release-on-tag workflow.
